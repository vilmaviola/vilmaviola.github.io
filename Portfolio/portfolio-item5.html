Text and speech mismatches in talking books

Speech Technology

The project was initiated by MTM The Swedish Agency for Accessible Media, a government agency that provides people with reading difficulties literature in accessible formats such as talking books.

The assignment

Our assignment was to find incorrect parts of talking books. We did this by looking for mismatches between the recorded version and the text version of the books. We used the software Aeneas, a forced alignment software which is a set of tools that automatically synchronize audio and text.

Implementation and solution

We created a program that combines the text files (HTML/Smil) with audio files (MP3), calculates the length for each sentence, decides an average reading speed of vowels per second, compare the sentences with the reading speed and lastly put red flags on the sentences that are far from the expected length.

The result

The output is a file showing the paragraph label at the top of each block of text so that the user easily can check in which paragraph of the book the error occurred. Then under the paragraph label the sentences that were red-flagged are displayed together with a number that indicates the difference in syllables per second, i.e. the speed of the narration. If the number is positive it means that the reading aloud rate is too slow which might indicate an insertion error, and if the difference is negative the reading aloud rate is too fast and this might indicate a deletion error.
