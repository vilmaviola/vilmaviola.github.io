AR shooting interface

Multimodal Interaction and Interfaces

We designed a prototype for an experiment of multimodal interaction. It was a prototype for Android devices and developed using the Unity game engine together with the Vuforia Unity package. The prototype has one UI layer with a gun that is always present on top of the camera feed. When the prototype detects the specified image target (an empty golden frame), a collision mesh is added on top of the frame. The mesh will detect collisions with other objects. The user can fire the gun by clicking on it, which will send out a paint-filled ball. When the ball hits a surface, it will splash. If it hits the frame, the collision mesh will detect the collision and Mona Lisa will be displayed in the frame.


The modalities we used were visual feedback, haptic feedback and auditory feedback. The visual feedback consists of the gun moving slightly upwards when a shot is fired, a splash of color when a bullet hits an object (wall or picture frame) and the frame being filled with the Mona Lisa painting when hit by a bullet. When the user fires the gun, the phone will vibrate. When the gun is fired, a gunshot sound is played. When a bullet hits the picture frame, a splash sound is played in sync with the splash visuals.


This was my first time using Unity and my role in the project was to implement the gun asset and to make the picture of Mona Lisa appear when the bullet hit the frame.
